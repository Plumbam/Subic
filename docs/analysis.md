# Анализ предметной области
  
## Введение

**Тема**: приложение для отслеживания платных подписок.
  
**Название**: Subic (тестовое название)
  
**Планируемые внешние интеграции**: сторонние базы данных с перечнем видов подписок и их характеристиками, подключение сервиса уведомлений.
  
**Описание**: ПОЛНОСТЬЮ ~~БЕС~~ПЛАТНОЕ мобильно приложение для отслеживания платных подписок на различных сервисах по типу Netflix, Яндекс Музыка, VK Музыка, Okko, PornHub, Onlifuns.  Данное приложение позволит контролировать расходы на подписки и удобно отслеживать их. Проанализировав рынок поняли, что в доступных приложениях интерфейс нас не устроил, а платить мы не согласны. Потому что подписка на подписки: «Ну это какой-то бред! Это кринж ребята». Главными особенностями будет бесплатный доступ и удобный нам интерфейс.

## Проблема и текущее состояние
  
Конкуренты в большинстве платные, а бесплатные варианты не оправдывают нашего времени и внимания. Плохой интерфейс.
  
### SubsCrab
  
Источник https://subscrabapp.ru

Сервис для отслеживания подписок.

- **Слабые стороны**: Интерфейс очень перегружен, плохо подобраны (в морской тематике). Платная подписка. Присутствует реклама.
  
- **Сильные стороны**: Большая база встроенных подписок. Удобная система отслеживания подписок в виде календаря. Возможность сканировать подписки из почты.

### Fleek
  
Источник https://lifehacker.ru/bobby 

Сервис для отслеживания подписок.
  
- **Слабые стороны**: Простой интерфейс, приложение для iOS, бесплатная версия имеет ограничения на количество отслеживаемых подписок.
- **Сильные стороны**: Позволяет добавлять подписки в уже существующий список, указывать название, сумму, дату и частоту платежей. Приложение предупреждает о предстоящих платежах. 

### Существующие проблемы и недостатки
  
Основные требования к искомому сервису следующие:

-	Бесплатновое
-	Хороший интерфейс
-	Календарь
-	Личный кабинет пользователя
  
Ни один из найденных сервисов не удовлетворят требования полностью.
  
## Цель и задачи проекта
  
Целью является проектирование и разработка сервиса для ведения списка фильмов, сериалов, игр, книг и комиксов.
  
Основные задачи:
-	Проанализировать предметную область.
-	Определить требования к проекту и сформировать техническое задание.
-	Спроектировать сервис.
-	Разработать MVP с подключением сторонних интеграций.
-	Провести ручное и автоматизированное тестирование сервиса.
-	Написать пользовательскую документацию.
-	Ввести сервис в эксплуатацию.




## Основные требования
  
### Функциональные требования
  
-	Система статусов подписок (заморожено, активно)
-	Регистрация и авторизация пользователей.
-	Поиск подписок в базе сервиса.
-	Интеграция со сторонними базами данных (база данных подписок).
-	Ведение статистики (количество подписок, потраченные деньги за весь период и за каждый месяц по отдельности).
  
### Нефункциональные требования
  
-	Минималистичный дизайн.
-	Наличие мобильной версии.
-	Хеширование паролей пользователей.
-	Сбор метрик работы сервиса.
-	Покрытие тестами.
-	Документация API.
-	Поддержка русского, английского и японского языков.
-	мобильная версия в виде PWA
